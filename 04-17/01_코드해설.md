### 셀 2 — init_db 함수 정의 [코드]
```python
import sqlite3
import os

DB_NAME = "test_sqlite3.db"

def init_db():
    if os.path.exists(DB_NAME):
        os.remove(DB_NAME)
    conn = sqlite3.connect(DB_NAME)
    cursor = conn.cursor()
    cursor.execute("""
        CREATE TABLE users (
            id       INTEGER PRIMARY KEY,
            name     TEXT NOT NULL,
            fullname TEXT
        )
    """)
    conn.commit()
    return conn
```
- **무엇을**: DB 파일을 지웠다가 다시 만들고, `users` 테이블을 생성한 뒤 연결 객체를 반환.
- **왜**: ① 시연마다 **깨끗한 상태**를 보장하려고 파일 삭제부터 한다. ② `CREATE TABLE` 은 한 번 실행 후에는 중복 생성 시 에러가 나므로 파일 삭제가 가장 단순한 초기화. ③ `commit()` 을 호출해 스키마 변경을 디스크에 확정한다.

### 셀 3 — Create 소개 (마크다운)
> "executemany()는 리스트 데이터를 한 번에 여러 행으로 삽입합니다."

- 다음 셀의 벌크 INSERT 를 예고.

---

### 셀 4 — 벌크 INSERT [코드]
```python
conn = init_db()
cursor = conn.cursor()

users_data = [
    ('spongebob', 'Spongebob Squarepants'),
    ('patrick',   'Patrick Star'),
    ('squidward', 'Squidward Tentacles'),
]

cursor.executemany(
    "INSERT INTO users (name, fullname) VALUES (?, ?)",
    users_data
)
conn.commit()
print(f"{len(users_data)}개의 사용자 데이터 삽입 완료.")
```
- **무엇을**: 3개의 튜플 리스트를 한 번의 `executemany` 로 삽입.
- **왜**: ① 루프 `for row in rows: cur.execute(...)` 보다 **드라이버가 배치로 처리**해 빠르다. ② `?` 바인딩으로 SQL 주입 원천 차단. ③ `commit()` 이 없으면 `INSERT` 는 트랜잭션에만 남고 **디스크에 기록되지 않는다**.

### 셀 5 — Read 소개 (마크다운)
> "fetchall()은 모든 행을 리스트로, fetchone()은 첫 번째 행 하나만 반환합니다. 결과는 튜플(tuple) 형식이며, 인덱스로 각 컬럼에 접근합니다."

- 반환 타입이 **튜플**이라는 점을 강조. dict 아님 — 다음 셀의 인덱싱 이해 기반.

---

### 셀 6 — 전체 조회 [코드]
```python
cursor.execute("SELECT id, name, fullname FROM users")
rows = cursor.fetchall()

for row in rows:
    print(f"ID: {row[0]}, Name: {row[1]}, FullName: {row[2]}")
```
- **무엇을**: 전체 행을 리스트로 받아 각 튜플의 인덱스로 출력.
- **왜**: `row[0]`, `row[1]` 인덱싱은 **컬럼 순서**에 의존하므로 `SELECT id, name, fullname` 처럼 순서를 **명시적으로** 지정하는 것이 안전하다. `SELECT *` 는 스키마 변경 시 순서가 뒤집힐 위험이 있다.

---

### 셀 7 — 조건 조회 + fetchone [코드]
```python
cursor.execute(
    "SELECT * FROM users WHERE name = ?",
    ('patrick',)   # 튜플 주의: 쉼표 필수
)
patrick = cursor.fetchone()
print(patrick)
```
- **무엇을**: `name='patrick'` 조건으로 한 행만 가져옴.
- **왜**: ① `fetchone()` 은 첫 행을 튜플로 반환, 결과가 없으면 `None`. ② 값 하나짜리 파라미터는 반드시 `('patrick',)` 처럼 **콤마 붙인 튜플**로 만들어야 한다 — `('patrick')` 은 그냥 문자열이므로 바인딩 시 타입 에러.

---

### 셀 8 — Update 소개 (마크다운)
> "UPDATE ... SET ... WHERE 구문으로 특정 행의 값을 수정합니다. WHERE 조건을 빠뜨리면 테이블 전체가 수정되니 항상 확인하세요."

- UPDATE 의 치명적 실수(WHERE 누락) 사전 경고.

---

### 셀 9 — UPDATE [코드]
```python
new_fullname = 'S. Squarepants'

cursor.execute(
    "UPDATE users SET fullname = ? WHERE name = ?",
    (new_fullname, 'spongebob')
)
conn.commit()

cursor.execute("SELECT * FROM users WHERE name = 'spongebob'")
print(cursor.fetchone())
```
- **무엇을**: 'spongebob' 한 행의 `fullname` 을 변경하고 즉시 재조회로 확인.
- **왜**: ① 수정 직후 `SELECT` 로 **사후 검증**하는 습관은 DB 작업의 기본. ② UPDATE 뒤 `commit()` 호출 필수 — 호출을 잊으면 다음 세션에 사라진다.

---

### 셀 10 — Delete 소개 (마크다운)
> "DELETE FROM ... WHERE로 조건에 맞는 행을 삭제합니다. WHERE 없이 실행하면 테이블의 모든 데이터가 삭제됩니다."

- 두 번째 경고. UPDATE 와 DELETE 는 동일한 위험 패턴이라 같은 톤으로 반복 강조.

---

### 셀 11 — DELETE [코드]
```python
cursor.execute(
    "DELETE FROM users WHERE name = ?",
    ('squidward',)
)
conn.commit()

cursor.execute("SELECT * FROM users")
for row in cursor.fetchall():
    print(row)
```
- **무엇을**: 'squidward' 삭제 후 전체 재조회.
- **왜**: DELETE 역시 `?` 바인딩으로 주입 차단. 실행 후 `fetchall()` 재조회로 **행 수 변화**를 확인하는 시연.

```python
def init_customer_db():

    cursor.execute("""
        CREATE TABLE customers (

            username TEXT NOT NULL UNIQUE,
            full_name TEXT
        )
    """)
    conn.commit()
    return conn

df = pd.read_csv(csv_url)
df_top10 = df.head(10)

    "INSERT INTO customers (username, full_name) VALUES (?, ?)",
    customer_data,
)
```
- **무엇을**: ① customers 테이블을 UNIQUE·AUTOINCREMENT 스키마로 생성. ② pandas로 CSV를 불러 상위 10개만 튜플 리스트로 바꾸고 executemany 로 일괄 삽입. ③ 이후 같은 셀에서 CUST_0002 UPDATE, CUST_0003 DELETE 까지 수행.
- **왜**:
  - `AUTOINCREMENT`: id를 수동 관리하지 않고 자동 증가. 삭제 후 번호를 재사용하지 않는다.
  - `UNIQUE username`: 같은 username 재삽입 방지 — 재실행 안전성 확보.
  - `df.head(10)`: 강의 시연 목적상 작은 샘플로 시간·출력을 통제.
  - `.apply(lambda row: (row['고객ID'], row['고객ID']), axis=1).tolist()`: DataFrame → 튜플 리스트로 변환하는 관용 패턴. `executemany` 의 입력 형식에 맞춘다.
  - `finally: conn.close()`: 예외가 나도 연결을 확실히 닫아 **파일 핸들 누수** 방지.
