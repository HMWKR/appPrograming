## 학습 목표

- `sqlite3.connect()` 와 `cursor()` 의 **역할 차이**를 말할 수 있다.
- `CREATE TABLE` 에서 `PRIMARY KEY / AUTOINCREMENT / NOT NULL / UNIQUE` 제약 조건을 선택·설명할 수 있다.
- **CRUD 4개 SQL 패턴**(`INSERT / SELECT / UPDATE / DELETE`)을 파이썬 코드로 실행할 수 있다.
- `?` **파라미터 바인딩**을 일관되게 사용하여 **SQL 주입**을 방지한다.
- `executemany` 로 리스트 데이터를 **벌크 INSERT** 한다.
- `fetchall()` / `fetchone()` 반환 형태(튜플)를 이해하고 인덱싱한다.
- `conn.commit()` 을 호출하지 않으면 **변경 사항이 디스크에 반영되지 않는다**는 것을 설명할 수 있다.
- CSV → DataFrame → SQLite 파이프라인을 구성할 수 있다(실습과제 1).
- 단일 파일 스크립트를 **여러 모듈(init/create/read/update/delete)** 로 분리해도 같은 시나리오가 동작함을 안다(실습과제 2).

## 핵심 개념

### 1) 파일 기반 DB (SQLite의 특성)

- SQLite3는 **별도 서버 없이 파일 하나**가 곧 데이터베이스다.
- `sqlite3.connect("xxx.db")` 는 파일이 없으면 **자동 생성**, 있으면 열어서 연결.
- Python **표준 라이브러리**에 포함 → `pip install` 불필요.
- 쓰임: 학습용, 프로토타입, 클라이언트 측 캐시, 단일 프로세스 앱.

### 2) Connection vs Cursor

- `conn = sqlite3.connect(...)` → DB **연결**을 나타내는 객체. `commit/close/rollback` 수행.
- `cur = conn.cursor()` → SQL 명령을 **실제로 실행**하는 작업 창구. `execute/executemany/fetchall/fetchone` 수행.
- 편의상 `conn.execute(...)` 직접 호출도 가능 (내부에서 임시 커서 생성).

### 3) CRUD 4대 연산

| 약어 | SQL | 예시 | 반드시 함께 | 위험 |
|---|---|---|---|---|
| C | INSERT | `INSERT INTO t (...) VALUES (?)` | `commit()` | UNIQUE 위반 |
| R | SELECT | `SELECT ... FROM t WHERE ...` | `fetchall()/fetchone()` | 없음 (읽기) |
| U | UPDATE | `UPDATE t SET ... WHERE ...` | `WHERE` + `commit()` | **WHERE 누락 = 전체 수정** |
| D | DELETE | `DELETE FROM t WHERE ...` | `WHERE` + `commit()` | **WHERE 누락 = 전체 삭제** |

### 4) 파라미터 바인딩 `?`

```python
# 안전: 드라이버가 값으로만 처리
cur.execute("SELECT * FROM users WHERE name = ?", ('patrick',))

# 위험: 문자열 포매팅은 SQL 주입을 허용한다
cur.execute(f"SELECT * FROM users WHERE name = '{user_input}'")
```

- 튜플 길이와 `?` 개수가 일치해야 한다.
- 값이 **하나**일 때도 `('patrick',)` 처럼 **콤마**를 붙여 튜플을 만들어야 한다. 없으면 문자열 취급 → 오류.

### 5) executemany — 벌크 INSERT

```python
rows = [('a', 'A'), ('b', 'B'), ('c', 'C')]
cur.executemany("INSERT INTO users (name, fullname) VALUES (?, ?)", rows)
```

- 파이썬 for 문 반복 `execute()` 보다 빠르다 (드라이버가 묶어서 처리).
- DataFrame을 `.apply(lambda row: (row['A'], row['B']), axis=1).tolist()` 로 바꿔 그대로 넣을 수 있다.

### 6) commit 의 의미

- `execute`/`executemany` 는 **트랜잭션**에 쌓아둘 뿐이다.
- `conn.commit()` 을 불러야 비로소 **디스크 파일에 확정 저장**된다.
- 프로그램이 commit 없이 종료되면 변경분은 **모두 롤백**된다.

### 7) WHERE 의 중요성

- UPDATE / DELETE 에서 `WHERE` 를 빠뜨리면 **테이블 전체**가 대상이 된다.
- 운영 DB에서는 반드시 `SELECT ... WHERE ...` 로 **영향 범위 선검증** 후 UPDATE/DELETE 를 실행한다.

### 8) 모듈화 (실습과제 2 맥락)

- 단일 스크립트가 커지면 **역할별 파일**로 나누어 재사용·가독성을 높인다.
- 본 노트북 기준 책임 분할:
  - `init.py` — DB 연결·테이블 준비
  - `create.py` — INSERT
  - `read.py` — SELECT
  - `update.py` — UPDATE
  - `delete.py` — DELETE
  - `main.py` — 위를 조합해 시연

## 용어 정리

| 용어 | 뜻 | 출처 (셀) |
|---|---|---|
| `PRIMARY KEY` | 각 행을 고유하게 식별하는 컬럼 | 2 |
| `AUTOINCREMENT` | 삽입 시 id를 자동 증가 | 13 |
| `NOT NULL` | 값이 반드시 있어야 함 | 2, 13 |
| `UNIQUE` | 해당 컬럼 값은 테이블 내 중복 불가 | 13 |
| `cursor` | SQL 실행을 담당하는 객체 | 2 |
| `commit()` | 변경사항을 디스크에 확정 저장 | 2, 4, 9, 11 |
| `fetchall()` | 모든 결과 행을 리스트로 반환 | 6 |
| `fetchone()` | 결과의 첫 행 하나를 튜플로 반환 | 7 |
| `executemany()` | 리스트를 받아 여러 행을 한 번에 삽입 | 4, 13 |
| `WHERE` | 행 필터. UPDATE/DELETE 에서는 필수 | 7, 9, 11 |
| 파라미터 바인딩 `?` | SQL 주입을 막는 안전한 값 전달 방식 | 7, 9, 11 |
