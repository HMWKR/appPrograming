# 제출 산출물 작성 절차

## 목적

앱프로그래밍 과제를 주차별 디렉토리와 실행 가능한 설명 노트북 형태로 정리하기 위한 절차입니다.  
각 산출물은 개인 과제 제출물처럼 보이도록 이름, 학과, 학년, 학번을 명확히 적고, 코드 셀과 설명 셀을 균형 있게 구성합니다.

## 기본 원칙

- 주차별 디렉토리 이름은 과제 주제와 날짜를 알아볼 수 있게 작성합니다.
- 노트북 파일명은 디렉토리명과 연결되도록 작성하고, 끝에 `_explained.ipynb`를 붙입니다.
- 코드는 노트북에서 처음부터 끝까지 실행되어야 합니다.
- 코드 바로 아래에는 왜 해당 코드가 필요한지 마크다운으로 설명합니다.
- 외부 링크, 원본 저장소 이름, 불필요한 참고 문구는 제출물에 노출하지 않습니다.
- 실행 중 생성되는 임시 파일은 제출 대상에서 제외합니다.

## 작성 흐름

1. 과제 시트에서 제출 대상 열과 주제를 확인합니다.
2. 각 주차별 디렉토리를 만들고 노트북 파일을 배치합니다.
3. 노트북은 제목, 학생 정보, 학습 목표, 코드, 설명, 정리 순서로 구성합니다.
4. 실습 주제가 웹 서비스나 화면 구조를 포함하면 `README.md`, Python 파일, `templates/`, `static/`처럼 필요한 보조 파일을 추가합니다.
5. 코드 셀은 작은 단위로 나누고, 각 셀이 어떤 입력을 받아 어떤 결과를 만드는지 설명합니다.
6. 마지막에는 결과 해석과 학습 내용을 짧게 정리합니다.

## 검증 흐름

1. 전체 노트북을 실행합니다.

```bash
jupyter nbconvert --to notebook --execute --inplace --ExecutePreprocessor.timeout=180 week*/**/*.ipynb
```

2. 노트북 내부에 error output이 남아 있는지 검사합니다.
3. Python 보조 파일은 `compile()` 또는 직접 실행으로 문법과 기본 동작을 확인합니다.
4. 외부 주소, 원본 저장소명, 협업 표현, 출처성 문구가 남아 있지 않은지 검사합니다.
5. `model.pkl`, `__pycache__`, `.ipynb_checkpoints` 같은 생성 파일이 포함되지 않았는지 확인합니다.
6. 최종 파일 목록과 README 매핑이 일치하는지 확인합니다.

## 제출 전 체크리스트

- 학생 정보가 README와 주요 노트북에 들어 있는가
- 주차별 디렉토리와 노트북 이름이 README 매핑과 일치하는가
- 모든 노트북이 처음부터 끝까지 실행되는가
- 노트북 내부 error output이 없는가
- 보조 Python 파일이 실행 가능한가
- 생성 파일과 캐시 파일이 제외되어 있는가
- 설명 문장이 개인 과제 제출 문체로 작성되어 있는가
